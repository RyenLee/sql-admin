# SQL Admin 本地调试指南

## 项目结构

```
sql-admin/
├── crates/
│   ├── api-types/        # 前后端共享 DTO（零后端依赖）
│   ├── frontend/         # Leptos WASM 前端
│   ├── interfaces/       # Axum HTTP 接口层（后端入口）
│   ├── application/      # 应用层（业务编排）
│   ├── domain/           # 领域层（核心业务逻辑）
│   ├── infrastructure/   # 基础设施层（持久化、加密、连接池）
│   └── shared/           # api-types 重新导出 + domain 桥接
├── scripts/              # 辅助脚本
├── data/                 # SQLite 数据库文件（运行时生成）
└── .env                  # 环境变量配置
```

## 环境要求

- Rust 1.95+（edition 2024）
- Node.js 18+（前端 CSS 构建）
- wasm-target：`rustup target add wasm32-unknown-unknown`
- Trunk：`cargo install trunk`

## 方式一：手动分步启动

### 1. 启动后端（终端 1）

```powershell
cd d:\Dev\workspace\2026\05\Rust\sql-admin

# 设置环境变量
$env:ENCRYPTION_KEY = "LiteAdmin2026DevEncryptionKey!!32b"
$env:DATABASE_URL   = "sqlite:data/sql_admin.sqlite3?mode=rwc"
$env:RUST_LOG       = "debug"

# 启动后端服务（监听 0.0.0.0:3000）
cargo run -p sql-admin-interfaces
```

后端启动成功后会输出：
```
Server running and ready to accept connections (0.0.0.0:3000)
```

### 2. 启动前端（终端 2）

```powershell
cd d:\Dev\workspace\2026\05\Rust\sql-admin\crates\frontend

# 安装前端依赖（首次或 package.json 变更后）
npm install

# 构建 Tailwind CSS
npm run build:css

# 启动前端开发服务器（监听 8080，自动代理 /api → localhost:3000）
trunk serve --port 8080
```

### 3. 访问应用

浏览器打开 http://localhost:8080

> 前端通过 Trunk proxy 将 `/api/*` 请求代理到后端 `localhost:3000`，无需硬编码后端地址。


```powershell

cargo build -p sql-admin-frontend --target wasm32-unknown-unknown
```


```powershell
# 验证 domain 不依赖 infrastructure，application 不依赖 interfaces
.\scripts\check_deps.bat
```

## 环境变量说明

| 变量 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `ENCRYPTION_KEY` | 是 | - | 密码加密密钥（32 字节） |
| `DATABASE_URL` | 是 | - | SQLite 连接字符串 |
| `SERVER_ADDR` | 否 | `0.0.0.0:3000` | 后端监听地址 |
| `RUST_LOG` | 否 | `debug` | 日志级别 |
| `APP_ENV` | 否 | `dev` | 环境（dev/prod） |
| `LOG_DIR` | 否 | 空（仅 prod 为 `./logs`） | 日志输出目录 |

也可通过项目根目录 `.env` 文件统一配置。

## 前端代理配置

前端 Trunk 开发服务器已配置代理（`crates/frontend/Trunk.toml`）：

```toml
[serve]
proxy = [
    { path = "/api", target = "http://localhost:3000" },
    { path = "/health", target = "http://localhost:3000" },
]
```

前端代码中使用相对路径 `/api` 发起请求，开发时由 Trunk 代理，部署时由 Nginx 等反向代理处理。

## 常见问题

### Q: 前端编译报错 `wasm32-unknown-unknown` not found

```powershell
rustup target add wasm32-unknown-unknown
```

### Q: 后端启动报错 `ENCRYPTION_KEY` not set

确保已设置环境变量或 `.env` 文件存在。PowerShell 中：
```powershell
$env:ENCRYPTION_KEY = "LiteAdmin2026DevEncryptionKey!!32b"
```

### Q: 修改 api-types 后前端未自动重编译

Trunk watch 默认忽略后端 crate 目录。api-types 变更需手动重启 `trunk serve`，或检查 `Trunk.toml` 的 `[watch]` 配置。

### Q: Tailwind 样式修改不生效

需重新执行 `npm run build:css`，或使用 `npm run watch:css` 监听模式。
